# Blommunity 사이트맵

> [feature-spec.md](feature-spec.md) 5.1~5.3의 화면 인벤토리를 트리로 펼친 문서입니다.
>
> 표기: 상태 열은 **화면 구현 기준**입니다 — ✅ 구현됨 / 🛠 일부 / 📅 예정.

세 개의 면이 있습니다([product-spec.md](product-spec.md) 4절).

```
운영자 어드민 (Admin)      — 5.1 — 우리가 운영

커뮤니티 기본 템플릿 (User) — 5.3 — 회원이 사용
```

## 1. 운영자 어드민 (5.1)

> 접근 전제: **모든 화면은 운영자(ADMIN) 토큰 필요** (로그인 제외). 역할 게이팅은 화면 안에서 추가로 적용됩니다.

```
운영자 어드민
├── 로그인 ............................ [공개]
├── 대시보드 .......................... [운영자]
├── 운영자 관리
│   ├── 운영자 목록 .................... [Member+]
│   ├── 운영자 상세 .................... [Member+]
│   └── 승인 / 권한 임명 / 삭제 ........ [Manager+ / Owner]
├── 사업자(테넌트) 관리
│   ├── 테넌트 목록 .................... [Member+]
│   └── 테넌트 상세 / 정지 / 삭제 ...... [Member+]
├── 감사 로그 뷰어 .................... [운영자]
└── 내 프로필 ........................ [본인]
```

| 화면 | 포함 기능 ID | 접근 권한 | 주요 컴포넌트 | 상태 |
|------|--------------|-----------|---------------|:----:|
| 로그인 | A-AC-02 | 공개(비인증) | 이메일·비밀번호 폼 | ✅ |
| 대시보드 | A-ST-01, A-ST-02 | 운영자 | 지표 카드(총 테넌트·사용자·게시물), 활성도 시계열 차트 | 📅 |
| 운영자 관리 | A-AM-01 ~ A-AM-07 | 조회 Member+ / 승인·삭제 Manager+ / 권한 임명 Owner | 목록+필터(전체/대기/승인), 상세, 승인·회수 버튼, 권한 임명 셀렉트 | ✅ |
| 사업자 관리 | A-TN-01 ~ A-TN-06 | Member+ (정지·삭제 포함) | 테넌트 목록, 상세, 정지/재개·삭제 액션, 사용량 위젯 | ✅ (정지·사용량 📅) |
| 감사 로그 뷰어 | A-AL-02 | 운영자 | 필터(시간순/운영자별/경로별), 로그 테이블, 상세 패널 | ✅ |
| 내 프로필 | A-AC-03, A-AC-04 | 본인 | 프로필 보기, 비밀번호 변경 폼 | ✅ |

> 권한 매트릭스(누가 무엇을 할 수 있는지) 전체는 [feature-spec.md](feature-spec.md) 1.2를 참고하세요.

## 2. 테넌트 콘솔 / CMS (5.2)

> 접근 전제: **회원가입/로그인 제외 전 화면 사용자(USER) 토큰 필요.** 콘솔은 주로 Owner/Manager 역할의 사용자가 씁니다. 같은 API가 사업자 자체 프론트에도 연동됩니다(🔌).

```
테넌트 콘솔
├── 회원가입 / 로그인 ................. [공개]
├── 대시보드 ......................... [사용자]
├── 공간 관리
│   ├── 공간 목록 / 생성 .............. [생성: 사용자 / 수정·삭제: Owner·Manager]
│   ├── 공간 멤버 ..................... [조회: 멤버 / 역할변경: Owner·Manager]
│   └── 공간 초대 ..................... [Owner·Manager]
├── 게시판 관리
│   ├── 게시판 목록 / 생성 ............ [생성: 공간 멤버 / 수정·삭제: Owner·Manager]
│   ├── 게시판 멤버 ................... [조회: 멤버 / 역할변경: Owner·Manager]
│   ├── 게시판 초대 ................... [Owner·Manager]
│   └── 카테고리 분류 ................. [Owner·Manager]
├── 회원 관리
│   ├── 회원 목록 / 상세 .............. [사용자]
│   └── 권한 조정 / 차단 / 비번 리셋 ... [Owner / Manager+]
├── 신고 / 모더레이션 ................. [Owner·Manager]
├── 사이트 설정 ...................... [Owner]
└── 내 프로필 ........................ [본인]
```

| 화면 | 포함 기능 ID | 접근 권한 | 주요 컴포넌트 | 상태 |
|------|--------------|-----------|---------------|:----:|
| 회원가입/로그인 | T-AC-01, T-AC-02 | 공개 | 테넌트+Owner 가입 폼, 로그인 폼(tenantId+username+password) | ✅ |
| 대시보드 | T-OP-04 | 사용자 | 인기 글·신규 가입·트래픽 카드 | 📅 |
| 공간 관리 | T-SP-01 ~ T-SP-15 | 생성: 사용자 / 수정·삭제·멤버역할: Owner·Manager | 공간 목록, 생성/수정 폼, 멤버 목록, 초대 생성·회수, 받은 초대 | ✅ (T-SP-13·14·15 📅) |
| 게시판 관리 | T-BD-01 ~ T-BD-16 | 생성: 공간 멤버 / 수정·삭제·멤버역할: Owner·Manager | 게시판 목록, 생성/수정 폼, 멤버 목록, 초대, 카테고리 | ✅ (T-BD-13~16 📅) |
| 회원 관리 | T-UM-01 ~ T-UM-07 | 조회: 사용자 / 권한: Owner / 삭제·리셋: Manager+ | 회원 목록·검색, 상세, 권한 셀렉트, 차단·비번 리셋 | ✅ (T-UM-06·07 📅) |
| 신고/모더레이션 | T-OP-01 ~ T-OP-03 | Owner·Manager | 신고 목록, 처리(무시/삭제/차단), 금칙어 등록 | 📅 |
| 사이트 설정 | T-ST-01 ~ T-ST-05 | Owner | 기본정보, 테마/디자인, 도메인 연결, 가입 정책, SEO 메타 | 📅 |
| 내 프로필 | T-AC-03 ~ T-AC-05 | 본인 | 프로필 수정, 비밀번호 변경, 계정 탈퇴 | ✅ |

## 3. 커뮤니티 기본 템플릿 (5.3)

> 접근 전제: 회원(USER) 시점. **회원에게 Blommunity는 보이지 않습니다** — 보이는 건 사업자의 커뮤니티뿐([user-spec.md](user-spec.md) 민수). 사업자는 이 템플릿을 쓰거나 API(🔌)로 자체 프론트를 만듭니다.

```
커뮤니티 (사업자 사이트)
├── 로그인 / 회원가입 ................. [공개]
├── 메인 (공간·게시판 목록) ........... [공개 / 멤버]
├── 게시판 상세 (글 목록) ............. [접근 권한에 따름]
│   └── 게시물 상세 ................... [접근 권한에 따름]
│       └── 댓글 / 대댓글 ............. [멤버]
├── 글쓰기 / 수정 .................... [게시판 멤버]
├── 받은 초대함 ...................... [본인]
├── 검색 결과 ........................ [접근 권한에 따름]
└── 내 프로필 / 설정 ................. [본인]
```

| 화면 | 포함 기능 ID | 접근 권한 | 주요 컴포넌트 | 상태 |
|------|--------------|-----------|---------------|:----:|
| 로그인/회원가입 | U-AC-01, U-AC-02 | 공개 | 가입 폼(tenantId 컨텍스트), 로그인 폼 | ✅ |
| 메인 (공간·게시판 목록) | T-SP-02, T-BD-02 | 공개/멤버 | 공간 카드 그리드, 게시판 목록, 접근 가능 항목만 노출 | ✅ |
| 게시판 상세 (글 목록) | U-PT-02 | 접근 권한에 따름 | 글 목록(페이지네이션), 정렬, 글쓰기 진입 버튼 | ✅ |
| 게시물 상세 | U-PT-03, U-CM-01~07 | 접근 권한에 따름 | 본문, 댓글·대댓글 트리, 댓글 작성 폼, (예정)반응 | ✅ (반응 U-CM-07 📅) |
| 글쓰기/수정 | U-PT-01, U-PT-04, U-PT-06 ~ U-PT-08 | 게시판 멤버 | 에디터, 이미지·파일 첨부, 태그 입력 | ✅ (첨부·태그 U-PT-06~08 📅) |
| 내 프로필/설정 | U-AC-03 ~ U-AC-05, U-NT-01~02 | 본인 | 프로필 수정, 비밀번호 변경, 탈퇴, 알림 목록·읽음 | ✅ (알림 U-NT 📅) |
| 받은 초대함 | U-PJ-03, U-PJ-04 | 본인 | 공간·게시판 초대 목록, 수락/거절 | ✅ |
| 검색 결과 | U-PT-12 | 접근 권한에 따름 | 검색 바, 결과 목록, 필터 | 📅 |

## 4. 공통 메모

- **접근 권한 표기**: "공개"=비인증 접근 가능, "사용자/운영자"=로그인 필요, "Owner·Manager"=해당 역할 이상, "본인"=자기 자신만. 비공개(PRIVATE) 공간/게시판은 멤버만 접근합니다([glossary.md](glossary.md) Visibility 참고).
- 모든 콘솔·커뮤니티 기능은 API로도 제공되므로(🔌), 사업자가 이 사이트맵을 참고해 자체 프론트를 설계할 수 있습니다([feature-spec.md](feature-spec.md) 5.4).
