> 실무형 문서. **무엇을, 어떻게 만들 것인가**를 정의합니다.
>
> "왜 만드는가"는 [제품 기획서]·[사용자 기획서]가 답합니다. 이 문서는 작업 단위로 기능을 쪼개고, 각 기능의 화면과 플로우를 정의합니다.

---

## 0. 이 문서를 읽는 법

### 0.1 구성
세 개의 사용자 그룹 × 도메인별로 기능을 정리합니다.

- **A — Admin (운영자)**: Blommunity 자체를 운영
- **T — Tenant (사업자)**: Blommunity를 빌려 자기 커뮤니티 운영
- **U — User (회원)**: 사업자의 커뮤니티에서 활동

### 0.2 표기 규칙

**기능 ID**: `[그룹][도메인 약자][번호]`
- 예: `T-SP-01` = Tenant / Space 도메인 / 1번 기능

**상태**
- ✅ 완료 (백엔드 구현됨)
- 🛠 진행/일부 구현
- 📅 계획됨
- 🔍 검토 필요

**우선순위**
- P1 — MVP 필수
- P2 — 운영 안정성
- P3 — 확장 기능

**제공 형태**
- 🔌 API — 백엔드 API로만 제공
- 🖥 화면 — 우리가 화면(콘솔/어드민)을 제공
- 📦 템플릿 — 커뮤니티 기본 템플릿에서 제공

## 1. Admin (운영자) 기능 명세

### 1.1 운영자 계정 (A-AC)

| ID | 기능 | 설명 | 우선순위 | 상태 | 제공 |
|----|------|------|----------|------|------|
| A-AC-01 | 운영자 가입 신청 | 새 운영자가 이메일·비밀번호로 가입 신청 | P1 | ✅ | 🖥 |
| A-AC-02 | 운영자 로그인 | 이메일·비밀번호 로그인, JWT 발급 | P1 | ✅ | 🖥 |

| A-AC-04 | 본인 프로필 보기 | 자기 정보 조회 | P1 | ✅ | 🖥 |

```
[가입 신청] → 상태: PENDING

[Manager 이상이 승인] → approvedAt 기록, 상태: APPROVED

```

## 2. Tenant (사업자) 기능 명세

> 사업자가 자기 커뮤니티를 만들고 운영하는 기능. 테넌트 콘솔(CMS) 화면에서 제공하며, 일부는 API만 제공해 사업자가 자기 프론트에 통합할 수 있게 합니다.

### 2.1 사업자 계정 (T-AC)

| ID | 기능 | 설명 | 우선순위 | 상태 | 제공 |
|----|------|------|----------|------|------|
| T-AC-01 | 테넌트 회원가입 | 테넌트 + 최초 Owner 사용자 동시 생성 | P1 | ✅ | 🖥🔌 |
| T-AC-02 | 사용자 로그인 | tenantId + username + password | P1 | ✅ | 🖥🔌 |
| T-AC-03 | 본인 프로필 수정 | username, email 변경 | P1 | ✅ | 🖥🔌 |
| T-AC-04 | 본인 비밀번호 변경 | 현재 비밀번호 확인 후 변경 | P1 | ✅ | 🖥🔌 |
| T-AC-05 | 본인 계정 탈퇴 | 소프트 삭제 | P1 | ✅ | 🖥🔌 |
| T-AC-06 | 토큰 갱신 (RefreshToken) | Access Token 재발급 | P1 | 📅 | 🔌 |

**플로우: 테넌트 회원가입**
```
[가입 폼: 테넌트명, username, email, password]
         ↓
[Tenant 생성 + 첫 User 생성 (role=OWNER)]
         ↓
[JWT 발급 → 즉시 로그인 상태]
```

## 3. User (회원) 기능 명세

> 사업자의 커뮤니티에 가입한 최종 이용자가 쓰는 기능. 기본 템플릿(📦) 또는 사업자의 자체 프론트엔드에서 API(🔌)를 통해 사용합니다. 회원에게 **Blommunity 자체는 보이지 않습니다.**

|----|------|------|----------|------|------|
| U-AC-01 | 회원가입 | tenantId 컨텍스트에서 username·password | P1 | ✅ | 🔌📦 |
| U-AC-02 | 로그인 | JWT 발급 | P1 | ✅ | 🔌📦 |
| U-AC-03 | 본인 프로필 수정 | username, email | P1 | ✅ | 🔌📦 |
| U-AC-04 | 본인 비밀번호 변경 | 현재 비밀번호 확인 | P1 | ✅ | 🔌📦 |
| U-AC-05 | 본인 계정 탈퇴 | 소프트 삭제 | P1 | ✅ | 🔌📦 |

## 6. 핵심 플로우 다이어그램

### 6.1 사업자의 첫 사용
```
[방문] → [테넌트 회원가입 T-AC-01]
       → [공간 생성 T-SP-01]
       → [게시판 생성 T-BD-01]
       → [(선택) 회원 초대 T-SP-07/T-BD-07]
       → [(선택) 자체 도메인 연결 T-ST-03]
       → 활동 가능
```

### 6.2 회원의 첫 활동
```
[방문] → [회원가입 U-AC-01]
       → [공간/게시판 둘러보기 T-SP-02 / T-BD-02]
       → [공개 가입 U-PJ-01/02] 또는 [초대 수락 U-PJ-04]
       → [게시물 작성 U-PT-01]
       → [댓글 U-CM-01]
```

### 6.3 운영자의 신규 가입 처리
```
[운영자 신규 가입 A-AC-01: status=PENDING]
        ↓
[기존 Manager+ 가 목록에서 확인 A-AM-01]
        ↓
[승인 A-AM-04: approvedAt 기록]
        ↓
[신규 운영자 로그인 가능]
```

### 6.4 데이터 접근 검증 (모든 요청 공통)
```
[요청] → [JWT 검증 X-SC-01]
       → [토큰 타입 검증 X-SC-02]
       → [역할 검증 X-SC-03]
       → [tenant_id 자동 주입 X-SC-07]
       → [Soft delete 필터 X-DT-01]
       → [Visibility / Membership 검증]
       → [응답]
```
